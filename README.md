# Ratevig Vault

Homelab automation for deploying a HashiCorp Vault instance on Proxmox.

The project provisions a Debian LXC container with Terraform, installs Vault with
Ansible, then configures Vault as an internal PKI and enables HTTPS.

## Repository layout

- `terraform/`: Proxmox LXC provisioning.
- `ansible/`: Vault installation, unseal, and configuration playbooks.
- `ansible/roles/unseal_vault/`: reusable role used to unseal Vault from stored unseal keys.

## Prerequisites

- Terraform.
- Docker, if you use `ansible/run-ansible.sh` as the Ansible control environment.
- SSH access to the target container as `root`.
- A Proxmox API token with permissions to create and start the LXC container.
- The SSH private key referenced by `ansible/ansible.cfg` inside the Ansible control environment:
  `/root/.ssh/ansible_id`.

## Secrets

Do not commit local plaintext secrets. The following files are expected to stay
private and untracked:

- `terraform/secrets.auto.tfvars`
- `ansible/.ansible.key`

`ansible/vault/secrets.yml` may be committed only if it is encrypted with
Ansible Vault. Never commit it in plaintext.

Example Terraform secrets file:

```hcl
pm_api_token_secret = "replace-me"
ct_password         = "replace-me"
ssh_key             = "ssh-ed25519 AAAA... user@host"
```

The Ansible Vault file must contain the Vault root token and the unseal keys
generated during the manual Vault initialization:

```yaml
vault_root_token: "replace-me"
vault_unseal_keys:
  - "replace-me"
  - "replace-me"
  - "replace-me"
```

Encrypt or edit it with Ansible Vault, using `ansible/.ansible.key` as the vault
password file.

## Installation

### 1. Provision the LXC container with Terraform

From the Terraform directory:

```bash
cd terraform
terraform init
terraform plan
terraform apply
```

The current Terraform configuration creates a Debian LXC container named
`vault-ct` on Proxmox with IP address `192.168.2.2/24`.

After the apply completes, verify that SSH works from the Ansible control
environment:

```bash
ssh root@192.168.2.2
```

### 2. Prepare the Ansible environment

From the repository root, enter the Ansible directory and install the required
collections:

```bash
cd ansible
ansible-galaxy collection install -r requirements.yml
```

If you were still in `terraform/` after the previous step, use `cd ../ansible`
instead.

If you use the provided Docker-based helper, run it from the Ansible directory:

```bash
./run-ansible.sh
ansible-galaxy collection install -r requirements.yml
```

The inventory currently targets:

```ini
[Vault]
vault ansible_host=192.168.2.2
```

### 3. Install Vault

Run the installation playbook:

```bash
ansible-playbook playbooks/install_vault.yml
```

This installs Vault on the Debian container, starts the service, and exposes the
initial Vault API over HTTP at:

```text
http://192.168.2.2:8200
```

### 4. Initialize Vault manually

Vault initialization is intentionally manual. From a machine that can reach the
Vault API:

```bash
export VAULT_ADDR=http://192.168.2.2:8200
vault operator init
```

Store the generated unseal keys and the initial root token securely. They are
required for the next Ansible steps.

### 5. Store the generated keys for Ansible

Create or update `ansible/vault/secrets.yml` with the values generated by
`vault operator init`:

```yaml
vault_root_token: "paste-root-token-here"
vault_unseal_keys:
  - "paste-unseal-key-1-here"
  - "paste-unseal-key-2-here"
  - "paste-unseal-key-3-here"
```

Encrypt the file with Ansible Vault:

```bash
ansible-vault encrypt vault/secrets.yml --vault-password-file .ansible.key
```

To edit it later:

```bash
ansible-vault edit vault/secrets.yml --vault-password-file .ansible.key
```

### 6. Unseal Vault

Run the unseal playbook:

```bash
ansible-playbook playbooks/unseal_vault.yml --vault-password-file .ansible.key
```

This uses `vault_unseal_keys` from `vault/secrets.yml` and checks that Vault is
unsealed.

### 7. Configure Vault with Ansible

Run the configuration playbook:

```bash
ansible-playbook playbooks/configure_vault.yml --vault-password-file .ansible.key
```

This playbook:

- installs Python dependencies required by the Vault modules;
- enables and configures the PKI secrets engine;
- generates the internal root CA if none exists;
- issues a Vault server certificate;
- switches Vault from HTTP to HTTPS;
- restarts and unseals Vault after the HTTPS configuration is applied.

After this step, Vault should be available at:

```text
https://192.168.2.2:8200
```

The playbook prints the generated root CA certificate. Trust this CA on client
machines that need to access Vault without certificate warnings.

## Common commands

```bash
# Terraform
cd terraform
terraform plan
terraform apply

# Ansible
cd ansible
ansible-playbook playbooks/install_vault.yml
ansible-playbook playbooks/unseal_vault.yml --vault-password-file .ansible.key
ansible-playbook playbooks/configure_vault.yml --vault-password-file .ansible.key
```
